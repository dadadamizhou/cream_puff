# 泡芙 - 高中英语抗遗忘跟读

## 1. 产品目标

面向高中生的移动端背词工具。默认采用全国高中英语 3500 词兼容模式，适配黑龙江哈尔滨学生；词库覆盖高一基础、高二进阶、高三冲刺和四级拓展，以“建立记忆、主动回忆、即时纠错、间隔复习”为核心闭环。

## 2. 核心用户流程

1. 用户注册或登录，账号之间学习数据完全隔离。
2. 首页查看今天的新学、到期复习、连续打卡与预计用时。
3. 用户可选择一个或多个学习词库；系统固定按高一、高二、高三、四级推进，在当前层级随机分配新词。
4. 完整流程先处理到期旧词：低阶段做英文到词义回忆，高阶段做中文到英文拼写。
5. 新词按每组 5 个学习；逐词听音、跟读、理解语境，可用触控或鼠标沿字形手写。
6. 每组学完立即遮住答案默写；答错时查看正确拼写后再次作答，连续不会可暂时跳过并进入 10 分钟重现。
7. 系统根据可判分结果或四档自评安排下一次复习；完成当天全部任务后形成打卡记录。
8. 主流程完成后进行全量混合巩固，通过选择、听力和默写抽查全部已学词。
9. 用户可随时发起自由复习，系统从已学且当前未到期的词中优先选择最久未复习的一组。
10. 完成今日必做任务后，用户可提前领取并学习明天的新词；提前任务不增加今天的必做目标。
11. 用户可在进度页查看本周完成度、累计掌握词数和最近学习记录。

## 3. 复习策略

- 默认每周新学 60 个词，可在设置中调整为 20 至 350 个。
- 系统按本周已过天数平滑分配新词；复习量由到期词数量自动决定。
- 新词只从用户启用的词库中分配；优先完成最低层级，同层使用数据库随机抽样，不按字母顺序出词。
- 词库采用成员关系而不是互斥标签；同一个词可以同时属于某个高中阶段和四级，用户只选四级时不会漏掉四级基础词，全量推进时由用户词卡唯一约束自动去重。
- 修改学习词库只影响后续新词分配，已学词和到期复习继续保留。
- 基础复习间隔：学习当天、1 天、2 天、4 天、7 天、15 天、30 天。
- 忘记：退回一个阶段，10 分钟后再次出现。
- 模糊：保持阶段，次日复习。
- 记得：进入下一阶段。
- 熟练：跨越两个阶段，减少无效重复。
- 当天没有完成的到期词自动顺延，不会丢失。
- 自由复习排除今天已复习和当前已到期的单词；记得或熟练不会推迟原复习计划，遗忘或模糊可将复习时间提前。
- 提前学习按上海时区计算明天的累计周目标，周日提前学习时自动进入下一周计划；重复领取不会重复分配。
- 当天完成至少一个学习任务即记录活跃；完成全部到期任务计为完整打卡。
- 新词首次学习不要求猜词义；先建立声音、含义、语境和拼写表征，再做遮挡回忆。
- 拼写任务不会自动播放发音；完成作答后才允许用发音核对，避免提示泄露。
- 手写用于建立字形和拼写线索，但不替代主动回忆与可判分的键盘默写。

> 首版使用可解释的艾宾浩斯间隔模型，不宣称严格复现某一条实验曲线。后续可基于用户遗忘率升级为个性化 FSRS 调度。

## 4. 功能范围

### 首版包含

- 邮箱 + 密码注册、登录、退出。
- HttpOnly 会话 Cookie 与账号数据隔离。
- 全国高中 3500 词兼容模式、高一/高二/高三主阶段与完整四级成员关系词库及可重复执行的种子导入。
- 今日任务生成、到期旧词优先、新词每 5 个分批学习与当批回测。
- 自由复习已学词、完成今日任务后提前学习明天新词。
- 发音播放、浏览器语音识别跟读、触控/鼠标手写练习。
- 四档掌握度反馈与服务端复习调度。
- 每日练习按已学词量动态安排 5 至 50 题：中选英、英选中、听音选义、听力默写、翻译默写；每个入选词优先覆盖两种不同题型，小词库最多各题型练习一次。
- 每日题单刷新可恢复，答题即时反馈；错词降低一个记忆阶段并在 10 分钟后复现。
- 今日/本周/累计进度、连续打卡。
- 每周新词目标、学习词库多选、浅色/深色/跟随系统主题。
- 中文界面，预留 `next-intl` 国际化结构。
- Neon PostgreSQL 表结构和版本化迁移。

### 首版不包含

- 短信、微信或学校统一身份认证。
- 云端语音评分；浏览器不支持识别时允许用户直接自评。
- 班级、教师后台、排行榜和社交关系。
- 离线下载、跨设备实时同步和推送通知。

## 5. 页面结构

- `/login`：登录与注册切换。
- `/study`：今日概览、当前学习任务和完成状态。
- `/practice`：每日一练概览、五种题型、即时反馈和完成成绩。
- `/progress`：周进度、掌握分布和最近学习数据。
- `/settings`：每周新词目标、学习词库、主题和账号操作。

移动端使用底部导航；桌面端限制内容宽度，保持与 H5 一致的任务流。

## 6. 数据模型

- `users`：账号、密码摘要、每周新词目标、启用词库、创建时间。
- `sessions`：不可逆会话令牌摘要、到期时间。
- `words`：单词、主学习阶段、词频位置、音标、释义和例句。
- `word_book_entries`：单词与高一、高二、高三、四级词库的成员关系和层内位置。
- `user_words`：用户对单词的阶段、到期时间、复习次数和掌握状态。
- `review_logs`：每次反馈、阶段变化、耗时和跟读结果。
- `daily_checkins`：每天完成数、目标数和是否完整打卡。
- `daily_checkin_words`：当天必做任务中已完成的去重单词，用于隔离自由复习和提前学习。
- `daily_practice_sessions`：用户每日动态练习场次、实际题量与成绩。
- `practice_questions`：每日题单、题型、选项、作答与判分结果。

## 7. 非功能要求

- 以 390px 宽度为主要设计基准，兼容 320px 至桌面宽度。
- 交互按钮最小触控高度 44px，支持安全区。
- 密码仅存储 bcrypt 摘要；会话令牌仅通过 HttpOnly Cookie 传输。
- 所有写接口用 Zod 校验；数据库查询必须带用户 ID。
- 首页核心内容直接访问 Neon PostgreSQL，不依赖第三方业务接口。
- 跟读、发音能力降级时不阻断背词主流程。

## 8. 首版验收标准

- 两个账号登录后看不到对方学习记录。
- 新用户登录后能自动获得当天任务并完成一轮学习。
- 自由复习不会增加今日打卡目标；提前学习不会混入今日必做任务，重复领取不会增加明日词数。
- 四种反馈会产生不同的下次复习时间和阶段变化。
- 五种练习题型均可完成；刷新后题单和已答结果保持不变。
- 两个账号无法读取或提交对方的练习题。
- 刷新页面或重新登录后进度保持。
- 数据库从空库执行迁移和种子后可正常启动，重复执行不会重复插入。
- `npm run lint`、`npm run test`、`npm run build` 通过。
